How the algorithm works -- Each file name is suffixed with a -A -B -C -S.
If there is not suffix, it gets defaulted to -D

These characters are rarity tiers. The algorithm then assigns weights to each file
and creates a giant list then picks a random number in that list to pick the trait.

The weights go as follows:
D - 40%
C - 30%
B - 25%
A - 4%
S - 1%

However, since tiers are not evenly distributed and there is no limit to how many of 
one trait can appear, there is a chance an S tier might show up more than 1% of the time.

I went ahead and let the algorithm do its work to decide what gets what.

After a few iterations, I was happy with the generations. Now, this list does not include
1 of 1s. The process for that included picking 10 random IDs within the first 1k items
and 8 IDs within the remaining 2k. Then after those were picked, there metaData were manually
added to the ipfs directory.
